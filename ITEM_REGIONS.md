# 物品识别区域配置

## 概述

物品识别区域配置用于定义游戏界面中物品槽的位置，支持对每个物品槽进行单独的 OCR 识别。

## 配置文件

配置文件：`items.json`

### 配置结构

```json
{
  "item_r0_c0": { "x": 480,  "y": 240, "width": 220, "height": 280 },
  "item_r0_c1": { "x": 720,  "y": 240, "width": 220, "height": 280 },
  ...
}
```

### 命名规则

物品区域命名遵循 `item_r{行号}_c{列号}` 的格式：
- `r` 表示行号（row），从 0 开始
- `c` 表示列号（column），从 0 开始

例如：
- `item_r0_c0` - 第 0 行，第 0 列
- `item_r1_c1` - 第 1 行，第 1 列

### 参数说明

| 参数 | 类型 | 说明 |
|------|------|------|
| x | int | 区域左上角的 x 坐标（相对于游戏窗口 client 区域） |
| y | int | 区域左上角的 y 坐标（相对于游戏窗口 client 区域） |
| width | int | 区域的宽度（像素） |
| height | int | 区域的高度（像素） |

## 使用方法

### 1. 自动创建

程序首次启动时，会自动创建 `items.json` 文件，包含默认配置。

### 2. 手动创建

从模板复制配置文件：
```bash
cp items.json.example items.json
```

### 3. 自定义配置

编辑 `items.json` 文件，根据实际游戏界面调整区域坐标：
1. 确保游戏窗口分辨率为 1920x1080
2. 使用屏幕截图工具测量物品槽位置
3. 更新对应的 x, y, width, height 值
4. 保存文件并重启程序

## 调试模式

在配置中启用调试模式可以查看详细的识别信息：

1. 打开设置窗口
2. 勾选"启用调试模式"
3. 保存设置

调试模式下，程序会在控制台输出：
- 识别的区域坐标
- 截图保存路径
- OCR 识别结果
- 识别到的文字内容

## 注意事项

- 所有坐标都是相对于游戏窗口 **client 区域**的
- 配置文件保存在项目根目录
- 不同游戏版本或分辨率可能需要调整配置
- 建议在标准分辨率（1920x1080）下配置

## 相关文件

- `items.json` - 物品区域配置文件（运行时生成）
- `items.json.example` - 配置文件模板
- `config.json` - 主配置文件
- `range.json` - 余额区域配置文件
